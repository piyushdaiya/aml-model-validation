# Narration / Talking Points

## Short Narration Script

This is the AML Model Validation Reporting Portal Demo.

Access is controlled through secure login and registration because the portal is designed for sensitive reporting and evidence review.

The executive dashboard stays shared, but shifts emphasis by persona for consulting and client stakeholders.

The validation inventory covers both traditional AML model validation reporting and GenAI-assisted AML workflow validation reporting in one place.

Traditional AML items keep familiar validation reporting sections.

GenAI workflow items extend the same reporting shell with workflow boundaries, grounding, response quality, safety and controls, testing evidence, findings, and report outputs.

The portal does not execute AML validation workflows in the consulting environment.

Instead, it provides secure access to findings, remediation, evidence summaries, and committee-ready audit packs.

That means consulting teams can provide reporting visibility across both traditional AML and GenAI assurance work while data and validation execution remain inside the financial institution.

## Optional Live-Talk Variant

- Secure access matters because the portal hosts sensitive reporting, not generic demo content.
- The same dashboard route adapts by persona for leadership, validation, compliance, and platform operations.
- The inventory stays shared across traditional AML and GenAI items.
- GenAI coverage focuses on reporting what matters: grounding, response quality, safety controls, findings, and audit-pack output.
- The core message is boundary discipline: secure reporting in the consulting environment, execution inside the institution.
